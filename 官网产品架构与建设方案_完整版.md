## 二、三方协同架构

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   官网本体    │ ←──→│  GEO 资产层   │ ←──→│  外部平台矩阵 │
│  (内容站)    │     │ (结构化数据)   │     │ (信任信号源)  │
└──────────────┘     └──────────────┘     └──────────────┘
  Astro 静态生成      JSON-LD Schema       美团/抖音/小红书/
  Tailwind + TS        + llms.txt          百度地图/视频号
                       + sitemap.xml
                       + robots.txt
```

官网是唯一权威数据源。`site.ts` 改一处全站生效。外部各平台的 NAP（名称、地址、电话）必须与 site.ts 完全一致。

## 三、GEO（生成式引擎优化）四层技术方案

### 第一层：基础设施（让 AI 能爬到）— 全部已落地 ✅

| 项目 | 实现 |
|------|------|
| 纯静态 HTML | Astro 默认输出，无客户端渲染依赖 |
| robots.txt 放行 AI | GPTBot / ClaudeBot / PerplexityBot / Baiduspider / Bytespider |
| sitemap.xml | ✅ |
| llms.txt | ✅（public/ 目录，Markdown 站点说明书） |
| 干净 URL | ✅（/nature-education/butterfly 等语义化路径） |
| ICP 备案 | ❌ 未启动，上线硬门槛（15-20 工作日） |

### 第二层：结构化数据（JSON-LD Schema）— 覆盖率 70%

| Schema | 页面 | 状态 |
|--------|------|------|
| TouristAttraction | 全站（BaseLayout） | ✅ |
| LocalBusiness | 首页/关于我们（BaseLayout） | ✅ |
| FAQPage + Question/Answer | FAQ 页（faq.astro 自动生成） | ✅ |
| Service + ItemList | 研学课程页（courses.astro） | ✅ |
| Article | 资讯活动页（news/） | 待填充内容后添加 |
| Event | 季节活动页 | 待填充内容后添加 |

### 第三层：内容写法规则（AI 摘要友好）

- **定义句前置**：每个板块第一句必须是自包含定义句
- **每个事实独立成段**：禁止跨段落指代（"详见上文""如上所述"）
- **多用表格 + 数字 + 列表**：AI 优先抓取结构化事实
- **标题用完整问句**：FAQ 已实现此格式，课程页和团建页同样适用
- **避免营销形容词**："震撼""绝美""必去"→ AI 判定低信息密度
- **价格必须给数字区间**：168-388 元/人 优于 "实惠价格"

### 第四层：外部信任信号（让 AI 相信你）

| 动作 | 说明 |
|------|------|
| NAP 一致性 | 美团/抖音/小红书/百度地图的地址电话与 site.ts 完全一致 |
| 外链建设 | 探店博主/媒体文章带上官网链接 |
| 资质公开 | 研学基地/科普基地证书扫描件 + 编号放入 about.astro |
| 社交账号填入 site.ts | 抖音/小红书/视频号 ID 填入后自动同步全站 |

## 六、FAQ 维护规则（已内置自动化）

- 所有 FAQ 内容在 `src/content/faq/*.md`
- 格式固定：`### 完整问句` + 答案段落
- `faq.astro` 自动从 Markdown 生成页面正文 + FAQPage JSON-LD
- 新增问答：在对应分类 .md 文件末尾追加即可，无需改任何代码
- 当前 6 个分类文件覆盖约 30 个问答，基本够用，上线后按实际咨询追加
